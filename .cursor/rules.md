# Project coding rules

This project must always follow these rules.

General philosophy:
- Write the simplest possible working code
- Keep files small
- Prefer clarity over cleverness
- Avoid abstractions unless absolutely necessary
- Avoid complex architecture
- No premature optimization

Code style:
- Minimal TypeScript
- Minimal Tailwind
- Prefer simple functions
- No deeply nested components
- No unnecessary hooks
- No unnecessary libraries

Structure rules:
- Keep folder structure simple
- One responsibility per file
- Avoid large utility layers

When generating code:
- Always choose the simplest implementation
- Prefer readability over patterns
- Avoid enterprise-style architecture
